Git

Git`as yra populiariausia kodo versijavimo programa.

Git komandos

  • init - sukuria git repositorija dabartiniame aplankale
  • add - prideda nurodytus failus prie commit'ui paruostu failu (stage`ina)
  • commit - m "zinute" - sukuria kodo versija is stage'intu failu ir prideda privalomaja zinute apie pakitimus (zinute turetu buti ne ilgesne nei 50 simboliu)
  • status - nurodo iformacija apie tavo lokalios git'o repozitorijos statusa
  • push - is lokalios repozitorijos issiuncia commit'us git i isorine repozitorija
  • pull - is isorines repozitorijos atsiunciami commit'ai i lokale repozitorija
  • clone - klonuoja nurodyta repozitorija is isorines repozitorijos
  • git log - parodo darytus commit'us. Iseiti is commit log Q raide.
  • git chechout branchName|commitID - copy paste coda is git log. Darbas tesis nuo pasirinkto commit'o.
  • branch naujosSakosPavadinimas - Sukuria nauja saka

Branch protection rules

Viena svarbiausiu saku apsaugos taisykliu yra Pull Request nustatymas.

Eiliskumas pull request

  1. Susikuriama nauja saka - git branch newBranchName
  2. Nueinana i nauja saka - git checkout newBranchName
  3. Atlieki uzduoti kuria apsiemeiai atlikti toje sakoje
  4. Nueiti i main saka - git checkout main
  5. Atjaujinti main saka - git pull
